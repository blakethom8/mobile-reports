What we built: A CLI tool (check_tennis.py) that logs into the Santa Monica recreation site, checks every preferred tennis court across Ocean View Park, Reed Park, and Memorial Park for a given date and time window, and reports availability sorted by location preference — all without opening a browser.

The Problem

The Santa Monica recreation department uses ActiveNet — a third-party recreation booking platform. Courts must be reserved 1–3 days in advance, with slots opening at 8:15 AM each morning. Blake's two preferred parks are Ocean View (closest, near Bay St) and Reed Park (7th & Wilshire). Checking availability manually requires logging in, navigating to each court individually, and reading a calendar — tedious when you want to scan 10+ courts across multiple days.

Updated Version — Automated Python Script

Playwright Headless Browser + CLI Interface

The final tool is a standalone Python script that runs a headless Chromium browser via Playwright. It logs in once, checks all courts in parallel navigation, filters by time window, and outputs a ranked report in under 30 seconds.

Architecture

1
Login once: Navigate to home, click Sign In, fill credentials, wait for authenticated session
2
Check each court by ID: Navigate directly to /reservation/landing/search/detail/{id}
3
Read calendar cells: Query .an-calendar-table-cell elements by aria-label prefix for the target date
4
Filter by time window: Parse slot text ("8:00 AM – 9:00 AM") and compare to requested window
5
Rank and display: Sort by park priority (Ocean View → Reed → Memorial), flag available courts prominently

CLI Usage

# Next 3 days, all time slots
python3 check_tennis.py

# Specific day and time window
python3 check_tennis.py --date thursday --time evening
python3 check_tennis.py --date tuesday --time morning
python3 check_tennis.py --date 2026-03-15 --time 7am

# Filter by park
python3 check_tennis.py --park reed
python3 check_tennis.py --park ocean-view

Key Technical Discoveries

Discovery 1 — The Search URL Filter

Blake spotted that the search page accepts a date/time filter as a URL-encoded JSON parameter (eventDateAndTime). This meant that in theory, a single search request could return all available courts for a given time window — eliminating the need to check courts one-by-one. While the final script still checks individual courts (more reliable for exact slot data), this discovery confirmed the approach was feasible.

The decoded filter structure: {"dateAndTime": [{"from_date_time": "2026-03-10 08:00:00", "to_date_time": "2026-03-10 09:00:00", "id": -1}], "isAllowFullDay": false, ...}

Discovery 2 — React SPA Requires JavaScript

The site is a React Single-Page Application. The raw HTML response is an empty shell — all calendar data is rendered client-side via API calls. A simple requests-based scraper retrieves only 50KB of boilerplate HTML with no useful court data. This ruled out lightweight scraping and required either Playwright (headless browser) or direct API reverse-engineering.

Discovery 3 — Java Backend with CSRF Protection

The backend uses Java (JSESSIONID-based sessions) with CSRF token protection via window.__csrfToken injected into the page. The Akamai CDN layer at akamai-anc.apm.activecommunities.com handles all REST API calls. Without a properly established JavaScript session, all /rest/ endpoints return 303 redirects regardless of CSRF token inclusion.

Discovery 4 — Calendar aria-labels Are Machine-Readable

Each calendar day cell has a clean aria-label attribute: "Mar 10, 2026 Fully booked" or "Mar 10, 2026 8:00 AM - 9:00 AM available 0 more availabilities". Once a headless browser renders the page, these are trivially parseable — no complex HTML parsing required.
